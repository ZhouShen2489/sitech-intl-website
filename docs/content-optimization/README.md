## Content Optimization Program

This folder defines the recurring content and UI optimization workflow for `sitech-intl-website`.

### Goal

Improve the public-facing website in controlled daily slices so the experience becomes:

- more modern and less rigid
- more credible in English and bilingual presentation
- cleaner in code structure and file usage
- easier to track over time

### Run Contract

Every scheduled run should:

1. Read [`SKILL.md`](/Users/zhoushen/Documents/sitech-intl-website/docs/content-optimization/SKILL.md).
2. Read [`site-directory.md`](/Users/zhoushen/Documents/sitech-intl-website/docs/content-optimization/site-directory.md).
3. Read and update [`progress-tracker.md`](/Users/zhoushen/Documents/sitech-intl-website/docs/content-optimization/progress-tracker.md).
4. Work on only a small batch of files each run.
5. Prefer high-impact customer-facing pages before lower-value cleanup.

### Definition Of Done For A File

A file can be marked `polished` only when:

- the UI or copy is materially improved
- the wording is reviewed for business accuracy and natural English
- any obviously dead code in that file is removed
- the affected site still builds or has a reasonable local validation step recorded

### Primary Principles

- Prioritize the Opera-first experience and shared customer-facing content.
- Avoid literal translation tone in English copy.
- Do not remove files unless they are verified unused.
- Favor simple maintainable changes over large speculative rewrites.
