# 宽带业务支撑平台

## 页面定位

宽带页面要讲清楚：这不是一个单独的装机工具，而是一套把宽带资源、业务开通、售后工单和运营分析串成闭环的平台。客户最关心的不是“有没有功能”，而是能不能更快开通、更稳维护、并且把过程看得更清楚。

## 目标客户

- 固定宽带团队、家庭宽带运营团队、装维与售后团队。
- 正在做宽带资源统一纳管、开通激活和工单联动的运营商团队。
- 想提升宽带开通效率、运维响应速度和用户感知质量的业务团队。

## PPT / 公司主张

- 宽带业务支撑平台面向宽带全生命周期管理。
- 涵盖宽带资源管理、业务开通激活、售后运维与运营分析等核心环节。
- 通过资源、流程、数据的统一管理，实现从装机受理到运行维护的全流程协同。
- 支持资源统一纳管与动态调配，减少人工核对与跨系统操作。
- 已在中国联通多个省公司落地，连接宽带资源、开通、工单和运营分析。

## 公开市场背景

- 宽带业务越来越像一个持续运营的问题，而不是单次装机问题。
- 买家通常关心四件事：资源是否准确、开通是否顺畅、工单是否闭环、运营是否可视。
- 这类页面可以用“resource to activation to service”来解释，不要只说“系统平台”。

## 网站内容角度

- 第一屏要直接讲出“让宽带资源、开通、工单和运营分析接成闭环”。
- 文案重点放在资源准确性、开通时长、装维协同和运维可视化。
- 避免写成笼统的“宽带系统介绍”，要让客户一眼看懂它解决哪一段流程。

## 推荐页面模块

- Hero：宽带业务支撑平台，讲清资源到开通再到运维的闭环。
- Buyer pain：资源分散、开通慢、运维不透明。
- Capability map：资源统一纳管、智能开通激活、工单与运营分析。
- Proof notes：中国联通多个省落地，宽带资源准确性 95%+。
- First scope：一个省公司、一个资源闭环、一个工单联动试点。
